Descrierea proiectului

"Să se conceapă un produs Web de tip PWA (Progressive Web Application) capabil să gestioneze timpul de efectuare a unor activități (tasks). Utilizatorii autentificați vor propune (tipuri de) activități (task-uri), asignându-le apoi unor persoane care trebuie să le execute într-o perioadă de timp specificată.

Printr-un URL anonim, informațiile despre un task ce trebuie efectuat pot fi expuse public.

Fiecare activitate va avea asociate informații privind demararea acesteia (interval orar, zona geografica, doar utilizatori calificați într-un domeniu specific etc.) și finalizarea (i.e. dovada realizării cu (in)succes a task-ului -- minimal, un URL indicând o resursă edificatoare: fotografie, video, depozit de cod-sursă, orice altceva).

Istoricul activităților și statistici vor fi disponibile tuturor celor implicați. Diversele notificări vor fi disponibile direct în browser și/sau trimise prin poșta electronică.

Inspirație: Asana și Trello."

Tehnologii utilizate

Pentru crearea interfeței utilizatorului, folosim limbajele HTML și CSS.

Pe partea de client, vom utiliza doar cod-sursă ECMAScript pur.
Pe partea de server, folosim Node.js și framework-ul Express.js. Am ales acest framework deoarece este minim,flexibil și oferă un set robust de funcții pentru web.

Stocarea și managementul datelor pe partea de server vor fi realizate utilizând un server de baze de date relațional, MySql. Stocarea datelor pe partea de client se face face utilizand Cache API.

Funcționalități ale aplicației

Pentru un utilizator care nu are cont singura funcționaliate va fi de a se înregistra.

Pentru un utilizator care are cont, funcționalitățile vor fi:

  • Posibilitatea de a se loga în aplicație
  • Posibilitatea de a se deloga din aplicație
  • Vizualizarea board-urilor din care face parte
  • Crearea unui board
  • Ștergerea unui board(doar daca el este cel care a creat acel board)
  • Vizualizarea taskurilor prezente
  • Adăugarea unui task
  • Ștergerea unui task
  • Editarea unui task
  • Asignarea de persoane pentru un task
  • Adăugarea unui deadline pentru un task
  • Posibilitatea adăugării de imagini, link-uri, depozit de cod sursă, zonă geografică, domeniu, descriere pentru fiecare task
  • Distribuirea unui task, în mod public, printr-un URL anonim
  • Adăugarea de etichete/categorii și filtrarea task-urilor în funcție de ele
  • Afișarea istoricului fiecărui board, pe baza caruia va primi notificări direct în browser sau prin email
  • Vizualizarea de statistici

Detalii despre proiect

Încadrarea aplicațieii într-o categorie de aplicație Web

Progresive Web Application(PWA)

Care sunt cerințele pentru a fi considerată o aplicație web progresivă:

  • Progresivă – Să funcționeze pentru orice utilizator, indiferent de browser-ul ales, deoarece este construit cu o îmbunătățire progresivă ca principiu de bază
  • Responsive – Să aibă un design potrivit pentru orice dispozitiv: desktop, mobil sau tabletă
  • Independentă de conectivitate – Să funcționeze în mod offline sau în rețele de calitate scăzută
  • Proaspătă – Întotdeauna actualizată

Single Page Application(SPA)

Care sunt cerințele pentru a fi considerată o aplicație SPA:

  • Vom folosi JavaScript care poate încărca conținut în document dacă și când utilizatorul are nevoie,fără a reîncărca întreaga pagină - aceasta este denumită în mod obișnuit Ajax.


  • Aceste api-uri vor trimite datele sub forma de fisiere JSON.

    API-uri pentru board-uri:

    • POST "/boards" -> va crea un nou board
    • GET "/boards" -> va oferi toate board-urile
    • GET "/boards/{id}" -> va oferi board-ul cu id-ul specificat
    • DELETE "/boards/{id}" -> va sterge board-ul cu id-ul specificat
    • PUT "/boards/{id}" -> care va face update unui board

    API-uri pentru task-uri:

    • POST "/tasks" -> va crea un nou task
    • GET "/tasks/{id_board}" -> va oferi toate task-urile unui board
    • DELETE "/tasks/{id}" -> va sterge task-ul cu id-ul specificat
    • PUT "/tasks/{id}" -> care va face update la task-ul cu id-ul dat ca parametru

    API pentru users:

    • GET "/users/{username}" -> va oferi detaliile despre un user

    API pentru istoric:

    • GET "/history/{id_board}" -> va oferi istoricul pentru un board dat ca parametru

    • Service worker

    Service Worker aparține aplicațiilor progresive web și este un script ce rulează în browser, în fundal. Service Worker cuprinde: Push Notification, sincronizare în fundal și posibilitatea de accesa un site pe care ai mai navigat, site pe care-l poți accesa chiar dacă ești deconectat de la Internet.

    Service Worker ne permite să rulăm paginile din cache-ul browserului, pentru a accesa conținutul dorit. Sincronizarea în fundal oferită de Service Worker permite să facem o acțiune chiar dacă suntem offline cu certitudinea că acesta va fi făcută când ne vom conecta din nou la Internet. De asemenea, paginile web pe care le accesăm și au Service Worker, pot fi accesate chiar și offline.
